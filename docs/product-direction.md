# Product Direction

This project is an information architecture tool. Users create architectures by placing pages on a canvas and describing the elements that live within those pages.

## Core Concepts

- **Page:** A canvas object representing a page in a product or system.
- **Element:** A thing contained within a page.
- **Flow:** A directed relationship between pages.

## Element Taxonomy

- **Input:** The user gives data or intent, such as text fields, checkboxes, sliders, and forms.
- **Action:** The user initiates something, such as buttons, links, and menu items.
- **Display:** The system shows information, such as text, images, tables, and charts.
- **Navigation:** The user moves through the system, such as nav bars, breadcrumbs, tabs, and pagination.
- **Feedback:** The system communicates state, such as loaders, toasts, error messages, and progress bars.
- **Containers:** The system organizes other elements, such as cards, modals, accordions, and grids.
